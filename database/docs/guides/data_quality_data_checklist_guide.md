# 실데이터 품질 점검 프롬프트 가이드

**시스템 역할:**  
당신은 WMS 창고관리시스템의 데이터 품질 점검 전문가입니다.

**사용자 요청:**  
아래의 SQL 파일 및 데이터베이스에 대해 실데이터 품질 체크리스트를 작성하고, 각 항목별로 PASS/FAIL 및 간단한 근거를 표로 작성해주세요.

**[SQL 파일 및 데이터베이스]**
- `database/schemas/wms/wms_ddl.sql`

- (운영/테스트 DB 접속 정보 등)

## 체크리스트 항목

### 1) 입력 데이터 유효성 검증
1. 프론트엔드/백엔드 모두에서 입력값(길이, 형식, 값의 범위, 필수값 등) 검증이 이루어지고 있는가?
2. 이메일, 전화번호, 사업자등록번호, 차량번호판 등은 정규표현식 등으로 형식 검증이 적용되어 있는가?

### 2) 데이터 무결성 검증
3. 중복 데이터가 발생하지 않도록 UNIQUE 제약조건 또는 중복 체크 로직이 적용되어 있는가?
4. 참조 무결성(FK)이 실제 데이터에 잘 반영되어 있는가? (연관 데이터 존재 여부 포함)

### 3) 데이터 품질 점검
5. 정기적으로 데이터 품질 리포트(예: NULL, 이상치, 코드값 불일치 등)를 생성하고 있는가?
6. 신규 데이터/마이그레이션 시 샘플 데이터를 추출하여 수동 검증을 수행하고 있는가?

### 4) 이상 데이터 탐지
7. 통계적 방법(평균, 최댓값, 최솟값 등)으로 이상치를 탐지하고 있는가?
8. 비즈니스 룰(예: 예약종료일 < 예약시작일, 견적금액이 음수 등)에 위배되는 데이터가 존재하지 않는가?

### 5) 테스트 데이터 관리
9. 운영 데이터와 테스트/스테이징 데이터가 분리되어 관리되고 있는가?
10. 테스트 데이터에 개인정보 등 민감 정보가 포함되어 있지 않은가? (익명화/마스킹 적용 여부)

### 6) WMS 창고관리시스템 특화 검증
11. 차량 번호판 형식이 표준에 맞게 입력되어 있는가?
12. 정비소 운영시간과 예약 시간이 일치하는가?
13. 견적 금액과 결제 금액이 일치하는가?
14. 사용자별 차량 소유 정보가 정확한가?

## 문서화 및 가이드 작성

- 검증 체크리스트, 예외처리 가이드, 품질 리포트 양식 등 표준 템플릿이 마련되어 있는가?
- 과거 데이터 오류 사례와 해결 방법이 문서화되어 있는가?
- 데이터 모델/품질 가이드가 주기적으로 리뷰 및 업데이트되고 있는가?

## 실무 예시

- "예약 종료일은 시작일보다 빠를 수 없습니다."  
  → CHECK (reservation_end_time >= reservation_start_time)
- "사용자 이메일은 반드시 고유해야 합니다."  
  → UNIQUE(email)
- "차량 브랜드는 코드테이블에 존재하는 값만 허용합니다."  
  → FK + vehicle_brand 테이블
- "정비소 사업자등록번호는 고유해야 합니다."  
  → UNIQUE(business_number)
- "예약 상태는 유효한 상태값만 허용합니다."  
  → FK + reservation_status 코드테이블

## WMS 창고관리시스템 도메인별 품질 점검

### 사용자 데이터 품질
- [ ] 사용자 전화번호 형식이 올바른가? (010-XXXX-XXXX)
- [ ] 사용자 이메일 형식이 올바른가?
- [ ] 사용자 주소 정보가 완전한가?

### 차량 데이터 품질
- [ ] 차량 번호판 형식이 표준에 맞는가?
- [ ] VIN 번호가 17자리로 입력되었는가?
- [ ] 차량 연식이 현재 연도보다 미래가 아닌가?

### 정비소 데이터 품질
- [ ] 정비소 전화번호 형식이 올바른가?
- [ ] 사업자등록번호가 10자리로 입력되었는가?
- [ ] 운영시간이 24시간 형식으로 입력되었는가?

### 예약/견적/결제 데이터 품질
- [ ] 예약 시간이 정비소 운영시간 내에 있는가?
- [ ] 견적 금액이 0보다 큰가?
- [ ] 결제 상태와 예약 상태가 일치하는가?
