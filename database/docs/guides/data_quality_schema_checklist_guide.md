# 데이터 품질 점검 프롬프트 가이드

## 개요
이 가이드는 WMS 창고관리시스템의 데이터 모델(ERD/DDL) 및 SQL 스키마 파일의 품질 점검을 위한 프롬프트 예시와 체크리스트 양식을 제공합니다.  
아래 프롬프트와 표를 활용하여 데이터 품질을 체계적으로 검증하세요.

---

## 프롬프트 예시

**시스템 역할:**  
당신은 WMS 창고관리시스템의 데이터 품질 점검 전문가입니다.

**사용자 요청:**  
아래의 SQL 파일들을 점검하여 데이터 품질 체크리스트를 작성하고, 각 항목별로 PASS/FAIL 및 간단한 근거를 표로 작성해주세요.

- `database/schemas/wms/vehicle_ddl.sql`

## 체크리스트 항목

### 1) 정합성(Consistency) 검증
1. ERD와 DDL의 컬럼, 데이터 타입, 제약조건(Primary Key, Unique, Not Null 등)이 모두 일치하는가?
2. 모든 컬럼의 데이터 타입이 WMS 창고관리시스템의 비즈니스 요구에 적합하게 정의되어 있는가?
3. NOT NULL, UNIQUE, DEFAULT 등 필수 제약조건이 누락된 곳은 없는가?
4. ERD/DDL 변경 시, 양쪽 모두 동기화가 이루어졌는가?

### 2) 관계(Reference) 검증
5. 모든 외래키(FK) 관계가 DDL에 명확히 정의되어 있는가?
6. ON DELETE/UPDATE 옵션이 WMS 창고관리시스템의 비즈니스 규칙에 맞게 설정되어 있는가?
7. ERD 상의 관계선이 실제 DDL의 FK와 일치하는가?

### 3) 코드/상태값 관리
8. 예약 상태, 결제 상태, 차량 브랜드 등은 코드테이블 또는 ENUM으로 관리되고 있는가?
9. 코드값이 하드코딩되어 있지 않고, 일관성 있게 관리되는가?
10. 코드테이블의 값이 실제 데이터와 일치하는가?

### 4) 이력/감사(History/Audit)
11. 주요 테이블에 created_at, updated_at, created_by, updated_by 컬럼이 모두 존재하는가?
12. 중요 데이터 변경 이력을 별도 테이블로 관리하고 있는가?
13. 이력 테이블의 구조와 활용 방법이 문서화되어 있는가?

### 5) WMS 창고관리시스템 특화 검증
14. 사용자, 차량, 정비소, 예약, 견적, 결제 등 핵심 도메인 테이블이 모두 정의되어 있는가?
15. 차량 정보(브랜드, 모델, 연식, 번호판, VIN)가 적절한 데이터 타입으로 정의되어 있는가?
16. 정비소 운영시간, 평점, 리뷰 등 정비소 특화 정보가 체계적으로 관리되고 있는가?
17. 예약 상태 이력, 견적 상태 이력 등 비즈니스 프로세스 이력이 적절히 관리되고 있는가?

## 결과 표 예시

| 체크리스트 항목 | PASS/FAIL | 근거/비고 |
|----------------|-----------|-----------|
| 1. 컬럼/타입/제약조건 일치 | PASS | wms_ddl.sql 모든 테이블이 ERD와 일치함 |
| 2. 데이터 타입 적합성 | FAIL | vehicle_ddl.sql의 price 컬럼이 VARCHAR로 정의되어 있음 (NUMERIC 권장) |
| 3. 제약조건 누락 | PASS | 모든 필수 컬럼에 NOT NULL, UNIQUE, DEFAULT 적용됨 |
| 4. ERD/DDL 동기화 | PASS | 최근 변경 이력 기준 동기화 완료 |
| 5. FK 정의 | FAIL | reservation_ddl.sql의 FK 제약조건이 누락된 컬럼 있음 |
| 14. 핵심 도메인 테이블 정의 | PASS | 사용자, 차량, 정비소, 예약, 견적, 결제 테이블 모두 정의됨 |
| 15. 차량 정보 데이터 타입 | PASS | 브랜드, 모델, 연식, 번호판, VIN 모두 적절한 타입으로 정의됨 |

## WMS 창고관리시스템 도메인별 품질 점검

### 사용자 관리 도메인
- [ ] 사용자 기본 정보와 프로필 정보의 관계가 적절히 정의되어 있는가?
- [ ] 사용자 인증 관련 정보가 보안을 고려하여 설계되어 있는가?

### 차량 관리 도메인
- [ ] 차량 브랜드와 모델의 계층 구조가 적절히 설계되어 있는가?
- [ ] 차량 소유자와 차량의 관계가 명확히 정의되어 있는가?

### 정비소 관리 도메인
- [ ] 정비소 운영시간과 서비스 유형이 체계적으로 관리되고 있는가?
- [ ] 정비소 평점과 리뷰 시스템이 적절히 연동되어 있는가?

### 예약/견적/결제 도메인
- [ ] 예약부터 결제까지의 전체 워크플로우가 테이블 구조에 반영되어 있는가?
- [ ] 각 단계별 상태 이력이 적절히 관리되고 있는가?
