# ERD 생성 가이드 프롬프트

## 개요
이 문서는 WMS 창고관리시스템의 데이터베이스 ERD(Entity Relationship Diagram)를 생성할 때 사용하는 프롬프트 가이드입니다. AI 도구나 데이터베이스 설계 도구에서 활용하여 일관성 있는 ERD를 생성할 수 있습니다.

---

## 🎯 ERD 생성 요청 프롬프트

```
다음 요구사항에 따라 WMS 창고관리시스템의 데이터베이스 ERD(Entity Relationship Diagram)를 생성해주세요:

## 시스템 개요
- **시스템명**: WMS 창고관리시스템 (WMS Management System)
- **목적**: WMS 창고관리 예약, 견적, 결제, 리뷰 등 통합 관리
- **데이터베이스**: PostgreSQL (wms 스키마)


## 시각적 요구사항
- 각 테이블은 도메인별로 색상 구분:
- 테이블명은 한글과 영문 병기
- 관계선은 화살표로 명확하게 표시 (1:N, N:1, 1:1)
- 테이블 크기는 컬럼 수에 비례하여 조정
- 도메인별로 논리적 그룹핑 및 배치

## 기술적 요구사항
- Draw.io 형식으로 생성
- 테이블은 swimlane 스타일로 구성
- 컬럼 정보는 상세하게 표시 (타입, 제약조건 포함)
- 관계선은 elbow 스타일로 깔끔하게 정리
- 인덱스 정보 포함 (PK, FK, UNIQUE 등)
- 테이블 코멘트 포함

이 요구사항에 따라 WMS 창고관리시스템의 전체적인 데이터 구조와 관계를 보여주는 ERD를 생성해주세요.
```


> 이 가이드를 활용하여 WMS 창고관리시스템의 ERD를 일관성 있고 체계적으로 생성하세요. 생성된 ERD는 데이터베이스 설계의 기초가 되므로 신중하게 검토하고 검증하는 것이 중요합니다.
