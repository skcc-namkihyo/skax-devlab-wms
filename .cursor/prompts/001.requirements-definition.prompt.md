# 올리브영 브랜드 파트너 플랫폼 - 요구사항정의서 작성 프롬프트

**⚠️ RFP 요구사항 문서 미 첨부시 작업을 중단하고 사용자에게 첨부요청, 첨부된 파일은 누락없이 모두 파악**

> [!CRITICAL]
> **Role**: Senior Business Analyst & System Requirements Expert
> **Goal**: RFP, 인터뷰 결과서, 현행시스템 분석서를 분석하여 **명확하고 구체적인 요구사항정의서**를 작성한다.
> **Constraint**: 추측이나 가정을 하지 않고 명확하지 않은 부분은 반드시 사용자에게 질의하여 확인 후 진행한다.
> {{ systemName }}: 파트너 시스템, 파트너 포털, 파트너 플랫폼 관리자 시스템, 파트너 플랫폼 전체공통, 글로벌 OMS, 브랜드원 시스템 기능 고도화 중 하나.
> {{ 1depthMenuName }}: 요구사항의 구조를 파악하여 1Depth 메뉴명을 한국어로 정의해야함. 예) 기준정보관리 
> {{ 2depthMenuName }}: 요구사항의 구조를 파악하여 2Depth 메뉴명을 한국어로 정의해야함. 예) 성분관리 
> {{ 3depthMenuName }}: 요구사항의 구조를 파악하여 3Depth 메뉴명을 한국어로 정의해야함(optional) 예) 전성분관리
> {{ 3digitSeqNum }}: 먼저 만들어진 파일명의 패턴을 파악하여 순번을 채번. 예) 001, 002, 003
> {{ fileName }}: {{ systemName }}-{{3digitSeqNum}}.{{ 1depthMenuName }}-{{ 2depthMenuName }}-{{ 3depthMenuName }}

## 필수 작업 계획 수립

- 다음 **3단계**로 작업을 분할하여 계획 수립, 파일에 기록되지 않는 단계별 작업 결과는 최대로 요약해서 표시(LLM 사용료 절약), 각 Task 는 사용자가 확인 후 실행(자동 실행 금지)

1. RFP 문서 분석 및 요구사항 파악
2. **추가 요구사항 수집 및 질의응답**
3. 요구사항정의서 작성 및 검증

## 실행 프로세스

### 1단계: 요구사항 파악(Critical)
**목표**: 요구사항 문서 확보 및 기능 범위 명확화

**수행 방법**:
1. 시스템 맥락 파악 및 대상 기능을 명확히 정의

### 2단계: 추가 요구사항 수집(Critical)
**목표**: 불명확한 요구사항과 구현 방식을 명확히 하기 위한 질의 수행
**요구사항정의서 저장 경로**: `docs/01.analysis/02.requirements/{{ fileName }}.md`

**수행 방법**:
1. **작업 중단**: 애매한 사항 발견 시 즉시 작업 중단
   - **⚠️ 추측이나 가정으로 진행하지 말고 반드시 명확한 답변을 받은 후 진행**
2. **질의 생성**: 프롬프트 채팅창에 구체적인 질문 작성
   - 질문은 최소 15개 이상, **많을수록 좋음**
   - **질문은 구체적이고 선택 가능한 옵션(A,B,C... 5개이상 선택)을 제시하여 작성**
   - **질의 대상 예시**:
     - **팝업(모달) 형태로 처리할지, 별도 페이지로 이동할지?**
     - 변경 이력관리가 필요한지?
     - 조회 이력관리가 필요한지?
     - 등록 페이지를 단일 페이지로 구성할지, 여러 단계로 분할할지?
     - 단계별 위자드 형태로 구현할지, 일반 폼 형태로 구현할지?
     - 임시 저장 기능이 필요한지, 한 번에 모든 데이터를 처리할지?
     - 특정 권한이나 조건이 필요한지?
3. **답변 대기**: 인간이 프롬프트로 답변을 할때까지 작업을 일시 중단
4. **요구사항 보완**: 답변을 바탕으로 요구사항 명확화

### 3단계: 요구사항정의서 작성 및 검증
**목표**: 사전 분석된 RFP 요구사항에 사용자의 요구사항을 반영하여 요구사항정의서 생성

**⚠️ 중요**: PST-EVA-PTE-001과 PST-EVA-PTE-002 시리즈의 RFP 문서들을 분석한 결과 기능적으로 중복이 발견되었습니다. 요구사항정의서 작성을 위해 먼저 다음 질문들에 대한 답변이 필요합니다.

## 성공 기준

**요구사항정의서 완성 체크리스트**:
- [ ] **헤더**: 버전, 담당자, 생성일자, 메뉴 경로 완비
- [ ] **개요**: 목적과 범위 명확 정의
- [ ] **기능 요구사항**: 모든 기능의 목적과 입력 요소, 비즈니스 규칙 정의
- [ ] **정책 및 규칙**: 사용자 질의응답을 통해 명확화된 모든 정책 반영
- [ ] **예외 처리**: 모든 예외 상황과 오류 메시지 정의
- [ ] **테스트 시나리오**: 정상 및 예외 시나리오 모두 작성
- [ ] **성능/보안**: 필수 요구사항 명시
- [ ] **데이터 모델**: 핵심 엔티티와 보관 정책 정의
