### 1단계: 프로젝트 설정 정보 질의 (필수)

**⚠️ 중요**: 이 프롬프트는 다양한 프로젝트에서 재사용 가능하도록 설계되었습니다. **프로젝트별 설정 정보를 사용자로부터 반드시 확인한 후** 변수를 치환하여 사용해야 합니다.

**질의 방법**:
1. 아래 "질의 항목" 섹션의 모든 항목을 사용자에게 질의합니다.
2. **사용자의 응답을 기다립니다. 응답이 없으면 작업을 진행하지 않습니다.**
3. 사용자로부터 모든 항목에 대한 응답을 받은 후, 응답 정보를 기반으로 프롬프트 내의 모든 `{변수명}`을 치환합니다.
4. 변수 치환이 완료된 후에만 다음 단계(작업 계획 수립)로 진행합니다.

**질의 항목**:

1. **프로젝트명**: 프로젝트의 전체 이름
   - 예: "WMS 창고관리시스템"
   - 변수: `{PROJECT_NAME}`

2. **프로젝트 간단명**: 프로젝트의 간단한 이름 (클래스명 생성에 사용)
   - 예: "WMS"
   - 변수: `{PROJECT_SIMPLE_NAME}` (PascalCase 변환: `WmsApplication`)
   - 변수: `{PROJECT_SIMPLE_NAME_LOWER}` (소문자, 하이픈: `wms`)

3. **기본 패키지명**: Java 기본 패키지명
   - 예: `com.execnt`
   - 형식: `com.[프로젝트간단명]` 또는 `[회사명].[프로젝트간단명]`
   - 소문자, 점(.)으로 구분, 특수문자 없음
   - 변수: `{BASE_PACKAGE}`
   - 변수: `{BASE_PACKAGE_PATH}` (점을 슬래시로 변환: `com/execnt`)

4. **데이터베이스 스키마명**: PostgreSQL 데이터베이스 스키마명
   - 예: `wms`
   - 소문자, 언더스코어(_) 허용, 특수문자 없음
   - 변수: `{DB_SCHEMA}`

5. **작성자명**: JavaDoc의 `@author` 태그에 사용할 작성자명
   - 예: "시스템"
   - 변수: `{AUTHOR}`

**질의 예시 메시지**:
```
프로젝트 설정 정보를 확인합니다. 아래 항목에 대해 답변해주세요:

1. 프로젝트명: [사용자 입력] (예: WMS 창고관리시스템)
2. 프로젝트 간단명: [사용자 입력] (예: WMS)
3. 기본 패키지명: [사용자 입력] (예: com.execnt)
4. 데이터베이스 스키마명: [사용자 입력] (예: wms)
5. 작성자명: [사용자 입력] (예: 시스템)

모든 항목에 대한 답변을 받은 후 작업을 진행하겠습니다.
```

**⚠️ 실행 규칙**:
- **사용자 응답 대기**: 질의 메시지를 전송한 후, 사용자의 응답이 올 때까지 **절대 다음 단계로 진행하지 않습니다.**
- **응답 확인**: 사용자로부터 모든 항목에 대한 응답을 받았는지 확인합니다.
- **응답 없음 처리**: 사용자 응답이 없거나 일부 항목이 누락된 경우, 작업을 중단하고 다시 질의합니다.
- **변수 치환**: 모든 응답을 받은 후, 아래 "변수 치환 규칙"에 따라 프롬프트 내의 모든 `{변수명}`을 실제 값으로 치환합니다.

**변수 치환 규칙**:
- 사용자로부터 입력받은 값을 프롬프트 내 모든 `{변수명}` 위치에 치환
- `{PROJECT_SIMPLE_NAME}` → PascalCase 변환 (공백 제거, 단어 첫 글자 대문자)
- `{PROJECT_SIMPLE_NAME_LOWER}` → 소문자 변환, 공백을 하이픈(-)으로 변환
- `{BASE_PACKAGE_PATH}` → `{BASE_PACKAGE}`의 점(.)을 슬래시(/)로 변환
- `{DATE}` → 현재 날짜 (YYYY-MM-DD 형식)

**프롬프트 내 변수 사용 규칙**:
- `{PROJECT_NAME}`: 프로젝트명 (예: "WMS 창고관리시스템")
- `{PROJECT_SIMPLE_NAME}`: 프로젝트 간단명 (예: "WMS")
  - 클래스명 생성 시 PascalCase로 변환 (예: "WMS" → `WmsApplication`)
- `{PROJECT_SIMPLE_NAME_LOWER}`: 프로젝트 간단명 소문자 (예: "wms")
  - application.yml의 `spring.application.name`에 사용
  - JWT secret 키에 사용
- `{BASE_PACKAGE}`: 기본 패키지명 (예: `com.execnt`)
  - 모든 Java 클래스의 패키지 선언에 사용
  - application.yml의 `mybatis.type-aliases-package`, `springdoc.packages-to-scan`에 사용
- `{BASE_PACKAGE_PATH}`: 기본 패키지 경로 (예: `com/execnt`)
  - `{BASE_PACKAGE}`의 점(.)을 슬래시(/)로 변환
  - 테스트 클래스 경로에 사용
- `{DB_SCHEMA}`: 데이터베이스 스키마명 (예: `wms`)
  - Mapper XML의 `FROM {DB_SCHEMA}.[table_name]`에 사용
- `{AUTHOR}`: 작성자명 (예: "시스템")
  - JavaDoc의 `@author` 태그에 사용
- `{DATE}`: 생성 날짜 (예: "2025-11-21")
  - 현재 날짜 사용 (YYYY-MM-DD 형식)
  - JavaDoc의 `@Date` 태그에 사용

### 2단계: DB 스키마 확인/생성
**목표**: DB 스키마 준비 (기존 활용 우선, 신규시 사용자 승인 필요)

**수행 방법**:
1. **스키마 구조 파악** (필수)
   - `database/schemas/*.sql` 파일 중 스키마 생성 관련 파일(`create_schema*.sql` 등)을 참조하여 데이터베이스 스키마 생성 및 구조 파악
   - 목적: 스키마 생성 및 구조 이해, `{DB_SCHEMA}` 스키마 사용 확인

2. **권한 구조 파악** (필수)
   - `database/schemas/*.sql` 파일 중 권한 관련 파일(`*role*.sql`, `*user*.sql`, `*permission*.sql`, `grant*.sql` 등)을 참조하여 역할(Role), 사용자(User), 권한 부여 구조 파악
   - 목적: 권한 구조 이해 및 적용

3. **테이블 구조 및 관계 파악** (필수)
   - `database/schemas/*.sql` 파일 중 테이블 생성 관련 파일(`create_tables*.sql`, `create_table*.sql` 등)을 모두 참조하여 테이블 구조와 관계 파악
   - 파일명 패턴 예시: `create_tables_phase*.sql`, `create_table_*.sql` 등
   - 각 Phase 파일은 기능 도메인별로 테이블을 구성하므로, 파일명과 내용을 분석하여 도메인별 데이터 모델 이해
   - 목적:
     - 기존 테이블 및 관계 활용 여부 판단
     - Phase별/도메인별 테이블 구조를 통한 기능 도메인별 데이터 모델 이해
     - 신규 테이블 필요시 해당 기능 도메인에 맞는 기존 파일에 추가하거나 신규 파일 생성 결정

4. **스키마 및 테이블 설계 규칙 준수** (필수)
   - **`.cursor/.rules/postgresql-standard-rule.mdc` 지침 완전 준수**:
     - 테이블/컬럼: `snake_case`, 단수형 (예: `user`, `vehicle`)
     - PK: `SERIAL` (PostgreSQL)
     - 공통 컬럼: `created_at`, `updated_at`, `created_by`, `updated_by`
     - 논리적 삭제: `is_active` (BOOLEAN, 기본값 TRUE)
     - 스키마: `{DB_SCHEMA}` 스키마 사용

5. **신규 테이블 생성** (필요시)
   - 신규 테이블 필요시: 해당 기능 도메인에 맞는 기존 파일에 추가하거나 신규 파일 생성
   - 스키마 DDL 작성 후 **사용자에게 승인 요청**

6. **DB-Java 매핑 확인** (필수)
   - DB 컬럼명(`snake_case`) ↔ MyBatis 자동 변환(`camelCase`) 매핑 확인 - `.cursor/.rules/backend.dev.instructions.mdc`의 "3.3. 데이터 처리 규칙" 참조
   - `application.yml`의 `map-underscore-to-camel-case: true` 설정 확인 (공통 모듈의 application.yml 참조)

```java
import {BASE_PACKAGE}.auth.filter.JwtAuthenticationFilter;
import org.springframework.context.annotation.Bean;

/**
 * Spring Security 설정을 담당하는 구성 클래스.
 * 
 * JWT 인증 필터, 보안 필터 체인, 세션 관리, CORS 설정 등의 보안 설정을 구성합니다.

 * @author {AUTHOR}
 * @Date {DATE}

@Configuration
@EnableWebSecurity
public class SecurityConfig {
    
    private final JwtAuthenticationFilter jwtAuthenticationFilter;
    
    public SecurityConfig(JwtAuthenticationFilter jwtAuthenticationFilter) {
        this.jwtAuthenticationFilter = jwtAuthenticationFilter;
    }
    
    /**
     * CORS 설정 소스 Bean
     * 
     * Frontend 애플리케이션과의 CORS 통신을 허용하기 위한 설정입니다.
     * 
     * @return CORS 설정 소스

        CorsConfiguration configuration = new CorsConfiguration();
        configuration.setAllowedOrigins(Arrays.asList(
            "http://localhost:3000",
            "http://localhost:4000",
            "http://localhost:5500",
            "http://127.0.0.1:3000",
            "http://127.0.0.1:4000",
            "http://127.0.0.1:5500"
        ));
        configuration.setAllowedMethods(Arrays.asList("GET", "POST", "PUT", "DELETE", "OPTIONS", "PATCH"));
        configuration.setAllowedHeaders(List.of("*"));
        configuration.setAllowCredentials(true);
        configuration.setMaxAge(3600L);
        
        UrlBasedCorsConfigurationSource source = new UrlBasedCorsConfigurationSource();
        source.registerCorsConfiguration("/api/**", configuration);
        return source;
    }
    
    /**
     * 보안 필터 체인 설정
     * 
     * Spring Security의 보안 필터 체인을 구성합니다.
     * CORS 설정, CSRF 비활성화, 세션 관리, JWT 인증 필터, 인가 규칙 등을 설정합니다.
     * 
     * @param http HttpSecurity 객체
     * @return SecurityFilterChain
     * @throws Exception 설정 중 발생할 수 있는 예외
     */
    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        http
            .cors(cors -> cors.configurationSource(corsConfigurationSource()))
            .csrf(csrf -> csrf.disable())
            .sessionManagement(session -> session.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
            .addFilterBefore(jwtAuthenticationFilter, UsernamePasswordAuthenticationFilter.class)
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/api/auth/**").permitAll()
                .requestMatchers("/api/health/**").permitAll()
                .requestMatchers(
                    "/swagger-ui.html",
                    "/swagger-ui/**",
                    "/v3/api-docs/**",
                    "/api-docs/**",
                    "/swagger-resources/**",
                    "/webjars/**"
                ).permitAll()
                .anyRequest().authenticated()
            );
        
        return http.build();
    }
}
```

```java
package {BASE_PACKAGE}.common.exception;

import {BASE_PACKAGE}.common.utils.ResponseUtil;
import io.swagger.v3.oas.annotations.Hidden;
import org.springframework.context.MessageSource;
import org.springframework.context.i18n.LocaleContextHolder;
import org.springframework.http.HttpStatus;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.ControllerAdvice;
import org.springframework.web.bind.annotation.ExceptionHandler;

import java.util.Map;

/**
 * 전역 예외 처리를 담당하는 핸들러 클래스.
 * 
 * 애플리케이션 전역에서 발생하는 예외를 처리하고 표준 형식의 에러 응답을 반환합니다.
 * BizException과 일반 Exception을 처리하며, 메시지 소스를 통해 다국어 메시지를 제공합니다.
 * 
 * @author {AUTHOR}
 * @Date {DATE}
 */
@ControllerAdvice
@Hidden
public class GlobalExceptionHandler {
    
    private final MessageSource messageSource;
    
    public GlobalExceptionHandler(MessageSource messageSource) {
        this.messageSource = messageSource;
    }
    
    /**
     * 비즈니스 예외 처리
     */
    @ExceptionHandler(BizException.class)
    public ResponseEntity<Map<String, Object>> handleBizException(BizException e) {
        String message = messageSource.getMessage(
            e.getErrorCode(), 
            null, 
            e.getMessage() != null ? e.getMessage() : "알 수 없는 오류가 발생했습니다.",
            LocaleContextHolder.getLocale()
        );
        
        return ResponseEntity
            .status(HttpStatus.BAD_REQUEST)
            .body(ResponseUtil.createErrorResponse(e.getErrorCode(), message));
    }
    
    /**
     * 일반 예외 처리
     */
    @ExceptionHandler(Exception.class)
    public ResponseEntity<Map<String, Object>> handleException(Exception e) {
        // Swagger 관련 예외는 제외 (Swagger가 자체적으로 처리)
        String className = e.getClass().getName();
        if (className.startsWith("org.springdoc") || className.startsWith("io.swagger")) {
            throw new RuntimeException(e); // Swagger 예외는 다시 던짐
        }
        
        // 개발 환경에서는 상세 에러 메시지 포함
        String errorMessage = "시스템 오류가 발생했습니다.";
        if (e.getMessage() != null) {
            errorMessage += " (" + e.getMessage() + ")";
        }
        e.printStackTrace(); // 스택 트레이스 출력
        return ResponseEntity
            .status(HttpStatus.INTERNAL_SERVER_ERROR)
            .body(ResponseUtil.createErrorResponse("E9999", errorMessage));
    }
}
```

**ResponseUtil.java** (응답 유틸리티):
```java
package {BASE_PACKAGE}.common.utils;

import java.util.HashMap;
import java.util.Map;

/**
 * API 응답 생성을 위한 유틸리티 클래스.
 * 
 * 성공 및 에러 응답을 표준 형식으로 생성하는 정적 메서드를 제공합니다.
 * 모든 API 응답은 이 유틸리티를 통해 일관된 형식으로 생성됩니다.
 * 
 * **API 응답 형식**:
 * - 성공 응답: `{result_code: "I0001", result_message: "...", data: {...}}`
 * - 에러 응답: `{result_code: "E1001", result_message: "..."}`
 * - **⚠️ 중요**: `success` 필드는 사용하지 않습니다. `result_code`로만 성공/에러 여부를 판단합니다.
 * 
 * @author {AUTHOR}
 * @Date {DATE}
 */
public class ResponseUtil {
    
    /**
     * 성공 응답 생성
     * 
     * @param resultCode 결과 코드 (예: "I0001", "I0002")
     * @param resultMessage 결과 메시지
     * @param data 응답 데이터 (선택사항)
     * @return 성공 응답 Map
     */
    public static Map<String, Object> createSuccessResponse(String resultCode, String resultMessage, Object data) {
        Map<String, Object> response = new HashMap<>();
        response.put("result_code", resultCode);
        response.put("result_message", resultMessage);
        if (data != null) {
            response.put("data", data);
        }
        return response;
    }
    
    /**
     * 성공 응답 생성 (데이터 없음)
     * 
     * @param resultCode 결과 코드 (예: "I0001", "I0002")
     * @param resultMessage 결과 메시지
     * @return 성공 응답 Map
     */
    public static Map<String, Object> createSuccessResponse(String resultCode, String resultMessage) {
        return createSuccessResponse(resultCode, resultMessage, null);
    }
    
    /**
     * 에러 응답 생성
     * 
     * @param errorCode 에러 코드 (예: "E1001", "E9999")
     * @param errorMessage 에러 메시지
     * @return 에러 응답 Map
     */
    public static Map<String, Object> createErrorResponse(String errorCode, String errorMessage) {
        Map<String, Object> response = new HashMap<>();
        response.put("result_code", errorCode);
        response.put("result_message", errorMessage);
        return response;
    }
}
```

**JwtUtil.java** (JWT 토큰 유틸리티):
```java
package {BASE_PACKAGE}.auth.util;

import io.jsonwebtoken.Claims;
import io.jsonwebtoken.Jwts;
import io.jsonwebtoken.security.Keys;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.stereotype.Component;

import javax.crypto.SecretKey;
import java.nio.charset.StandardCharsets;
import java.util.Date;
import java.util.HashMap;
import java.util.Map;

/**
 * JWT 토큰 생성 및 검증을 담당하는 유틸리티 클래스.
 * 
 * Access Token과 Refresh Token을 생성하고, 토큰의 유효성을 검증하며,
 * 토큰에서 사용자 정보를 추출하는 기능을 제공합니다.
 * 
 * @author {AUTHOR}
 * @Date {DATE}
 */
@Component
public class JwtUtil {
    
    private final SecretKey secretKey;
    private final long accessTokenExpiration;
    private final long refreshTokenExpiration;
    
    public JwtUtil(
        @Value("${jwt.secret}") String secret,
        @Value("${jwt.access-token-expiration}") long accessTokenExpiration,
        @Value("${jwt.refresh-token-expiration}") long refreshTokenExpiration
    ) {
        this.secretKey = Keys.hmacShaKeyFor(secret.getBytes(StandardCharsets.UTF_8));
        this.accessTokenExpiration = accessTokenExpiration;
        this.refreshTokenExpiration = refreshTokenExpiration;
    }
    
    /**
     * Access Token 생성
     */
    public String generateAccessToken(Long userId, String email, String userType) {
        Map<String, Object> claims = new HashMap<>();
        claims.put("userId", userId);
        claims.put("email", email);
        claims.put("userType", userType);
        
        return Jwts.builder()
            .claims(claims)
            .subject(String.valueOf(userId))
            .issuedAt(new Date())
            .expiration(new Date(System.currentTimeMillis() + accessTokenExpiration))
            .signWith(secretKey)
            .compact();
    }
    
    /**
     * Refresh Token 생성
     */
    public String generateRefreshToken(Long userId) {
        Map<String, Object> claims = new HashMap<>();
        claims.put("type", "refresh");
        
        return Jwts.builder()
            .claims(claims)
            .subject(String.valueOf(userId))
            .issuedAt(new Date())
            .expiration(new Date(System.currentTimeMillis() + refreshTokenExpiration))
            .signWith(secretKey)
            .compact();
    }
    
    /**
     * 토큰 검증
     */
    public boolean validateToken(String token) {
        try {
            Jwts.parser()
                .verifyWith(secretKey)
                .build()
                .parseSignedClaims(token);
            return true;
        } catch (Exception e) {
            return false;
        }
    }
    
    /**
     * 토큰에서 Claims 추출
     */
    public Claims getClaimsFromToken(String token) {
        return Jwts.parser()
            .verifyWith(secretKey)
            .build()
            .parseSignedClaims(token)
            .getPayload();
    }
    
    /**
     * 토큰에서 사용자 ID 추출
     */
    public Long getUserIdFromToken(String token) {
        Claims claims = getClaimsFromToken(token);
        return Long.parseLong(claims.getSubject());
    }
    
    /**
     * 토큰 만료 여부 확인
     */
    public boolean isTokenExpired(String token) {
        try {
            Claims claims = getClaimsFromToken(token);
            return claims.getExpiration().before(new Date());
        } catch (Exception e) {
            return true;
        }
    }
}
```

**JwtAuthenticationFilter.java** (JWT 인증 필터):
```java
package {BASE_PACKAGE}.auth.filter;

import {BASE_PACKAGE}.auth.util.JwtUtil;
import jakarta.servlet.FilterChain;
import jakarta.servlet.ServletException;
import jakarta.servlet.http.HttpServletRequest;
import jakarta.servlet.http.HttpServletResponse;
import org.springframework.security.authentication.UsernamePasswordAuthenticationToken;
import org.springframework.security.core.authority.SimpleGrantedAuthority;
import org.springframework.security.core.context.SecurityContextHolder;
import org.springframework.security.web.authentication.WebAuthenticationDetailsSource;
import org.springframework.stereotype.Component;
import org.springframework.web.filter.OncePerRequestFilter;

import java.io.IOException;
import java.util.Collections;
import java.util.List;

/**
 * JWT 토큰 기반 인증을 처리하는 필터 클래스.
 * 
 * HTTP 요청의 Authorization 헤더에서 JWT 토큰을 추출하고 검증하여,
 * SecurityContext에 인증 정보를 설정합니다.
 * 
 * @author {AUTHOR}
 * @Date {DATE}
 */
@Component
public class JwtAuthenticationFilter extends OncePerRequestFilter {
    
    private final JwtUtil jwtUtil;
    
    public JwtAuthenticationFilter(JwtUtil jwtUtil) {
        this.jwtUtil = jwtUtil;
    }
    
    @Override
    protected void doFilterInternal(
            HttpServletRequest request,
            HttpServletResponse response,
            FilterChain filterChain
    ) throws ServletException, IOException {
        
        String requestPath = request.getRequestURI();
        logger.debug("JwtAuthenticationFilter - Processing request: " + requestPath);
        
        // Authorization 헤더에서 토큰 추출
        String authHeader = request.getHeader("Authorization");
        logger.debug("JwtAuthenticationFilter - Authorization header: " + (authHeader != null ? "present" : "null"));
        
        if (authHeader == null || !authHeader.startsWith("Bearer ")) {
            logger.debug("JwtAuthenticationFilter - No valid Authorization header, continuing filter chain");
            filterChain.doFilter(request, response);
            return;
        }
        
        String token = authHeader.substring(7); // "Bearer " 제거
        logger.debug("JwtAuthenticationFilter - Extracted token: " + token.substring(0, Math.min(20, token.length())) + "...");
        
        try {
            // 토큰 검증
            if (jwtUtil.validateToken(token) && !jwtUtil.isTokenExpired(token)) {
                logger.debug("JwtAuthenticationFilter - Token is valid");
                
                // 토큰에서 사용자 정보 추출
                Long userId = jwtUtil.getUserIdFromToken(token);
                io.jsonwebtoken.Claims claims = jwtUtil.getClaimsFromToken(token);
                String email = (String) claims.get("email");
                String userType = (String) claims.get("userType");
                
                logger.debug("JwtAuthenticationFilter - User authenticated: userId=" + userId + ", email=" + email + ", userType=" + userType);
                
                // 권한 설정 (필요시)
                List<SimpleGrantedAuthority> authorities = Collections.singletonList(
                    new SimpleGrantedAuthority("ROLE_" + (userType != null ? userType : "USER"))
                );
                
                // SecurityContext에 인증 정보 설정
                UsernamePasswordAuthenticationToken authentication =
                    new UsernamePasswordAuthenticationToken(
                        userId,
                        null,
                        authorities
                    );
                authentication.setDetails(new WebAuthenticationDetailsSource().buildDetails(request));
                SecurityContextHolder.getContext().setAuthentication(authentication);
                
                logger.debug("JwtAuthenticationFilter - Authentication set in SecurityContext");
            } else {
                logger.debug("JwtAuthenticationFilter - Token validation failed or expired");
            }
        } catch (Exception e) {
            // 토큰 검증 실패 시 로그만 남기고 계속 진행
            logger.debug("JWT token validation failed: " + e.getMessage(), e);
        }
        
        filterChain.doFilter(request, response);
    }
}
```

#### Frontend 리소스 분석 및 API 연동 수정
**⚠️ 중요**: Backend API 개발 완료 후 Frontend 리소스를 분석하여 Mock 데이터를 실제 API 호출로 전환해야 합니다.

**Frontend 리소스 분석 절차**:
1. **Frontend 개발 결과 문서 확인**: `docs/03.development/[입력된 UI 설계서 폴더명].frontend.dev-result.md` 파일에서 다음 정보 추출

   - 사용된 Mock 데이터 파일 경로 (`frontend/api/v1/[menu-name].json`)
   - API 호출 패턴 (직접 axios 호출 또는 useCrud 사용)
   - 라우터 설정 (`frontend/router.js`)

2. **Frontend 소스코드 분석**:
   - **View 컴포넌트 분석**: `frontend/views/[menu-name]/[Page].js` 파일들을 읽어 다음 확인
     - Mock 데이터 사용 여부 (`import` 문 또는 직접 axios 호출)
     - API 엔드포인트 패턴 (현재 Mock 경로 또는 실제 API 경로)
     - 요청/응답 데이터 구조
   - **Composables 분석**: `frontend/composables/` 디렉토리의 파일들 확인
     - `useCrud.js`: Mock 데이터 경로와 API 엔드포인트 매핑 확인
     - `useAuth.js`: 인증 API 호출 패턴 확인
   - **API 모듈 분석**: `frontend/app.js` 또는 `frontend/api.js` 확인
     - Axios 인터셉터 설정 (JWT 토큰 자동 추가)
     - `baseURL` 설정 (`http://localhost:8080/api`)
     - 에러 처리 로직 (401 에러 시 로그인 페이지 리다이렉트)

3. **API 엔드포인트 매핑 테이블 작성**:
   - Mock 데이터 경로 → Backend API 엔드포인트 매핑
   - 예시:
     | Frontend Mock 경로 | Backend API 엔드포인트 | HTTP Method | 설명 |
     |-------------------|----------------------|-------------|------|
     | `frontend/api/v1/users/profile.json` | `/api/users/profile` | GET | 프로필 조회 |
     | `frontend/api/v1/users/profile.json` | `/api/users/profile` | PUT | 프로필 수정 |
     | `frontend/api/v1/vehicles.json` | `/api/users/vehicles` | GET | 차량 목록 조회 |
     | `frontend/api/v1/vehicles.json` | `/api/users/vehicles` | POST | 차량 등록 |

4. **Frontend 소스코드 수정**:
   - **View 컴포넌트 수정**: Mock 데이터를 실제 API 호출로 전환
     - 직접 axios 호출인 경우: `axios.get('http://localhost:8080/api/[endpoint]')` 형식으로 수정
     - `useCrud` 사용인 경우: `mockDataPath` 파라미터를 `null`로 설정, `apiEndpoint`는 `/api/[resource]` 형식으로 전달
   - **Composables 수정**: `frontend/composables/useCrud.js`에서 Mock 데이터 제거
     - `mockDataPath` 파라미터를 `null`로 설정
     - `apiEndpoint`는 `/api/[resource]` 형식으로 전달 (내부에서 자동 정규화)
   - **API 모듈 확인**: `frontend/app.js` 또는 `frontend/api.js`의 설정 확인
     - `baseURL`: `http://localhost:8080/api` 확인
     - Axios 인터셉터: JWT 토큰 자동 추가 및 401 에러 처리 확인
   - **라우터 확인**: `frontend/router.js`에서 라우트 설정 확인 (필요시 수정)

5. **CORS 설정 확인**: Backend의 `SecurityConfig.java`에서 `CorsConfigurationSource` Bean이 설정되어 있고, Frontend 포트(`http://localhost:4000`, `http://localhost:3000` 등)가 허용되어 있는지 확인

**⚠️ 주의사항**:
- Frontend 소스코드 수정 전 **반드시 사용자에게 확인 요청**
- 공통 파일(`app.js`, `router.js`, `store.js` 등) 수정 시 Git 충돌 가능성 고려
- 여러 개발자가 같은 파일을 작업하면 Git 충돌이 발생하므로, 공통 파일 수정 전 사용자 확인 과정을 거치고 작업 결과서 파일에 기입하여 공통 코드 담당자가 알 수 있게 합니다.

#### 통합 테스트 수행
- **Backend 서버 실행**: `./gradlew bootRun` 또는 IDE에서 실행

- **인증 플로우 테스트**:
  - Frontend 로그인 페이지에서 로그인 시도

  - 수정: 수정 페이지 접근, 데이터 수정 및 저장 확인
  - 삭제: 삭제 버튼 클릭 및 확인 다이얼로그 후 삭제 확인
- **에러 처리 테스트**:
  - 유효성 검증 오류: 잘못된 입력 시 에러 메시지 표시 확인
  - 인증 오류: 토큰 만료 시 로그인 페이지로 리다이렉트 확인
  - 서버 오류: 500 에러 발생 시 사용자 친화적 메시지 표시 확인

**사용 방법**:
1. **프로젝트 설정 질의**: 프롬프트 실행 전 "프로젝트 설정 (필수 질의)" 섹션의 항목을 사용자에게 질의하여 변수 값 확인
2. **변수 치환**: 사용자로부터 입력받은 값을 프롬프트 내 모든 `{변수명}` 위치에 치환
   - `{PROJECT_NAME}` → 프로젝트명
   - `{PROJECT_SIMPLE_NAME}` → 프로젝트 간단명 (PascalCase 변환)
   - `{PROJECT_SIMPLE_NAME_LOWER}` → 프로젝트 간단명 소문자 (하이픈 변환)
   - `{BASE_PACKAGE}` → 기본 패키지명
   - `{BASE_PACKAGE_PATH}` → 기본 패키지 경로 (점을 슬래시로 변환)
   - `{DB_SCHEMA}` → 데이터베이스 스키마명
   - `{AUTHOR}` → 작성자명
   - `{DATE}` → 현재 날짜 (YYYY-MM-DD)
3. **공통 모듈 생성**: 프롬프트의 "공통 모듈" 섹션 코드를 변수 치환 후 동일하게 생성
4. **도메인 모듈 생성**: Task 정의서 기반으로 생성하되, rule 파일의 공통 규칙 준수 및 변수 치환 적용
5. **Swagger 구성**: 프롬프트의 Swagger 상세 구성 가이드 참조하여 동일한 구조와 상세도 유지
6. **Backend 개발 결과 문서 생성**: 5단계에서 `backend.dev-result.md` 파일 필수 생성
7. **Frontend-Backend 연동**: 6단계에서 Mock 데이터를 실제 API로 전환하여 통합 테스트 수행
