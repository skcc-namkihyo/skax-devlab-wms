# /gen-task

## 개요
기능 요구사항을 정형화하고 작업 단위로 분해하는 교육용 명령어.
WMS 도메인 문맥에서 입고, 출고, 재고, 이동 등의 기능을 체계적으로 분석합니다.
분석 작업만 수행하며 코드 생성은 하지 않습니다.

## 입력
- 기능명 (예: "입고 관리", "출고 관리")
- 옵션: 추가 요구사항 설명

## 워크플로우

> **Skills**: task-template (Task 정의서 8-Section 템플릿 자동 로드)

1. **요구사항 정형화**
   - 기능의 핵심 목표 정의
   - 주요 사용자/역할 식별
   - 필수 속성(attribute) 목록화

2. **기능 분해 트리**
   - 상위 기능 → 중위 기능 → 하위 기능 (3단계)
   - 예: 입고 관리 → 입고 등록 → 상품 추가 / 입고 검수 → 수량 확인

3. **Task 목록 생성**
   - DB Task: 엔티티 설계
   - BE Task: API 설계 + 구현
   - FE Task: UI 설계 + 구현
   - 각 Task마다 예상 난이도(Easy/Medium/Hard) 표시

4. **산출물 저장**
   - task-{feature}.md 파일 생성
   - 마크다운 형식: 헤더, 테이블, 코드블록

## 산출물
- **task-{feature}.md**: 요구사항 분석 문서
  - 기능 개요
  - 요구사항 목록 (테이블)
  - 기능 분해 트리 (다이어그램 or 리스트)
  - Task 분해 (테이블: Task ID, 계층, 난이도, 설명)

## 체크포인트
- [ ] 기능명이 명확한가?
- [ ] 분해된 Task가 각 계층(DB/BE/FE)을 모두 커버하는가?
- [ ] 각 Task의 난이도가 현실적인가?
- [ ] 의존성 관계가 명확한가?
