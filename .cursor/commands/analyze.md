# /analyze

## 개요
실무 환경에서 기존 코드베이스를 고려한 요구사항 분석을 수행합니다.
/gen-task의 확장 버전으로, 기존 코드 탐색을 포함합니다.

## 입력
- 기능명 또는 변경 사항 설명
- 예: "입고 관리 개선", "출고 프로세스 재설계"

## 워크플로우

### Step 1: 요구사항 정형화
- 비즈니스 목표 정의
- 이해관계자(Stakeholder) 식별
- 필수/선택 기능 분류
- 제약조건 및 가정(Assumptions) 정리

### Step 2: 기존 코드베이스 탐색
- 관련 엔티티 조회 (DB, Entity class)
- 기존 API 엔드포인트 확인
- 관련 컴포넌트 검색
- 유사 기능 분석 (재사용 가능성)

### Step 3: 기능 분해
- 기존 구조와의 일관성 검토
- 재사용 범위 식별
- 신규 개발 범위 명확화
- Task 의존성 그래프

### Step 4: 문서화
- analysis-{feature}.md 생성
- 요구사항 매트릭스 (기능 × 우선순위)
- 기존 코드 레퍼런스 (파일 경로)
- 변경 영향도 예측

## 산출물
- **analysis-{feature}.md**
  - 요구사항 요약
  - 기존 코드 참조 (링크)
  - 기능 분해 다이어그램
  - Task 테이블 (우선순위 포함)

- [ ] 재사용 가능한 컴포넌트를 식별했는가?
- [ ] Task 의존성이 명확한가?
- [ ] 영향도 범위가 명시되었는가?
